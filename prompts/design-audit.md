# 디자인 감사 체크리스트

> 17개 카테고리 기반 디자인 품질 평가.
> 100점 시작, Critical -8 / Warning -4 / Tip -1.

## 점수 기준

| 점수 | 판정 |
|------|------|
| 90~100 | 출시 가능 |
| 70~89 | 소폭 수정 |
| 50~69 | 보완 필요 |
| 50 미만 | 재작업 |

### 1. 타이포그래피

- [ ] 폰트 최대 2종 사용 (`PretendardGOV` + `JetBrains Mono`)
- [ ] 계층 명확 -- h1 > h2 > h3 > body 크기 차이 유지
- [ ] 줄 간격 1.5~1.75 (본문), 1.2~1.4 (제목)
- [ ] 줄 길이 65~75자 이내 (본문 컬럼)
- [ ] 좌측 정렬 원칙 (RTL 제외)
- [ ] 텍스트 대비율 4.5:1 이상

### 2. 색상 & 대비

- [ ] 모든 텍스트: 4.5:1 이상 (AA 기준)
- [ ] UI 요소(버튼 보더, 아이콘): 3:1 이상
- [ ] 색맹 안전 -- 색상만으로 상태 표현하지 않음
- [ ] 팔레트 일관성 -- 토큰 외 하드코딩 없음
- [ ] 다크 모드 대응 시: 반전이 아닌 재설계

### 3. 간격 & 레이아웃

- [ ] 8pt 그리드 기반 간격 사용
- [ ] 관련 요소 근접, 무관 요소 충분한 여백
- [ ] 패딩 일관성 (컴포넌트 내부 토큰 사용)
- [ ] 텍스트/아이콘 정렬 축 일치

### 4. 시각 위계

- [ ] Primary 액션이 가장 눈에 띔
- [ ] F-패턴 또는 Z-패턴 읽기 흐름 지원
- [ ] 크기/대비/간격이 중요도 순서와 일치

### 5. 일관성

- [ ] 같은 목적의 컴포넌트가 동일한 스펙 사용
- [ ] 아이콘 패밀리 1종만 사용
- [ ] Corner Radius가 토큰 스케일 준수
- [ ] 중첩 반경 규칙 적용 (내부 = 외부 - padding)

### 6. 접근성

- [ ] 터치 타겟 44x44px 이상 (모바일)
- [ ] 모든 인터랙티브 요소에 Focus 상태 존재
- [ ] 폼 라벨 연결 (visible label, not placeholder only)
- [ ] 에러 메시지가 색상 외 텍스트로도 전달
- [ ] 읽기 순서가 시각 순서와 일치

### 7. 폼 & 입력

- [ ] 라벨이 필드 위 또는 좌측 (플레이스홀더 대체 금지)
- [ ] 입력 높이 최소 40px
- [ ] 필수 필드 명시 (`*` + 범례)
- [ ] 유효성 검사 타이밍: blur(형식) / submit(필수)
- [ ] 에러 메시지: 필드 하단 인라인, "무엇이 잘못됐는지 + 어떻게 고치는지"

### 8. 모션 & 애니메이션

- [ ] 애니메이션이 목적 있음 (장식용 과다 사용 금지)
- [ ] 지속 시간 150~500ms 범위
- [ ] Ease Out (등장) / Ease In (퇴장) 이징 적용
- [ ] `prefers-reduced-motion` 대응 명시

### 9. 다크 모드

- [ ] 라이트 모드 단순 반전이 아닌 별도 설계
- [ ] 표면 깊이 표현 (그림자 → 레이어 밝기 차이로 대체)
- [ ] 다크 모드에서 채도 감소 적용
- [ ] 토큰 기반으로 테마 스위칭 구현

### 10. 반응형 디자인

- [ ] 모바일 퍼스트 설계
- [ ] 360 / 768 / 1280px 브레이크포인트 테스트
- [ ] 가로 스크롤 없음 (모바일)
- [ ] 본문 폰트 최소 16px (모바일)
- [ ] 테이블: 모바일에서 가로 스크롤 또는 카드 변환 처리

### 11. 상태 (States)

- [ ] Loading 상태 설계 (Skeleton / Spinner / ProgressBar)
- [ ] Empty 상태: 아이콘 + 헤드라인 + CTA 포함
- [ ] Error 상태: 수준별 처리 (필드 인라인 / 페이지 레벨)
- [ ] Success 상태: Toast 또는 인라인 확인 메시지
- [ ] Disabled 상태: Opacity 40~60% + cursor 변경

### 12. 마이크로카피

- [ ] 버튼 레이블: 동사형
- [ ] 에러 메시지: 3파트 공식 (무엇이 / 왜 / 어떻게)
- [ ] 플레이스홀더: 라벨 대체 금지, 예시값만
- [ ] 용어 일관성: 같은 개념에 같은 단어 사용
- [ ] 톤앤매너 일관성 (공식적/친근한 레벨 통일)

### 13. i18n & RTL

- [ ] 하드코딩된 문자열 없음 (다국어 대응)
- [ ] 텍스트 확장 버퍼 30~50% 여유 (한 → 영 확장 시)
- [ ] RTL 지원 필요 시: 레이아웃 미러링 설계
- [ ] 날짜/숫자/통화 형식: 로케일 감지 후 적용

### 14. 엘리베이션 & 그림자

- [ ] 그림자가 토큰 스케일 준수 (sm / md / lg / xl)
- [ ] 엘리베이션 높을수록 그림자 강도 증가
- [ ] 다크 모드: 그림자 대신 레이어 밝기 차이 사용

### 15. 아이콘그래피

- [ ] 아이콘 라이브러리 1종만 사용
- [ ] 아이콘 크기: 16 / 20 / 24 / 32 / 40 / 48px (8pt 그리드)
- [ ] Outline/Filled 스타일 혼용 금지
- [ ] 인터랙티브 아이콘 터치 타겟 44x44px 이상
- [ ] 모호한 아이콘에 레이블 또는 툴팁 필수
- [ ] `aria-label` -- 아이콘 버튼 전체 적용

### 16. 내비게이션 패턴

- [ ] 현재 위치가 명확히 표시 (Active State)
- [ ] 탭 ≠ 최상위 내비 (탭은 같은 맥락 내 병렬 뷰)
- [ ] 3레벨 이상 깊이: 브레드크럼 제공
- [ ] 모바일 하단 내비: 3~5개 항목 + 레이블 필수
- [ ] 뒤로가기: URL 히스토리가 아닌 앱 흐름 기반
- [ ] Skip Navigation 링크 (접근성)

### 17. 디자인 토큰 & 변수

- [ ] 색상 하드코딩 없음 (토큰 사용)
- [ ] 간격 하드코딩 없음 (토큰 사용)
- [ ] 색상 토큰 연결 확인
- [ ] 다크 모드 토큰 스위칭 구현 확인
- [ ] 시맨틱 네이밍 준수 (예: `surface/primary` not `#FFFFFF`)

## Quick 5 -- 납품 전 최소 점검

모든 납품에서 반드시 확인하는 5가지 항목.

- [ ] 텍스트 대비율 4.5:1 이상
- [ ] 인터랙티브 요소에 Hover/Focus 상태 존재
- [ ] 하드코딩 값 없음 (토큰 사용)
- [ ] 모바일 터치 타겟 44px 이상
- [ ] Loading/Empty/Error 상태 설계 완료
