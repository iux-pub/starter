# 프로젝트 프로파일

KRDS 체크리스트는 모든 프로젝트에 같은 방식으로 적용하지 않는다. 코드 생성 전 프로젝트 유형을 먼저 판정하고, 공통 품질 규칙과 조건부 정부/공공 규칙을 분리한다.

## 프로파일

| 프로파일 | 적용 대상 | 기본 생성 | 조건부 생성 |
|----------|-----------|-----------|-------------|
| `private-corporate` | 민간 기업, 브랜드, 캠페인, 일반 웹사이트 | 시맨틱 구조, 접근성, 토큰, 반응형 | 정부 상징/공식 배너/운영기관 식별자 제외 |
| `admin-cms` | CMS, 관리자, 사내 업무 시스템 | 고밀도 정보 구조, 폼/테이블/검색 패턴, 접근성 | 정부 아이덴티티 제외. 공공 운영 CMS라면 화면 성격별로만 적용 |
| `public-affiliated` | 공기업, 출자·출연기관, 산하기관, 공공 서비스 | 공통 품질 + 공공 톤의 헤더/푸터 | 공식 배너/운영기관 식별자는 사업 요구와 기관 정책이 확인된 경우만 |
| `public-government` | 중앙부처·지자체·정부 상징 사용 서비스 | 공통 품질 + KRDS 공공 아이덴티티 후보 | 공식 배너, 정부 상징, 운영기관 식별자, 공공 푸터 필수 정보 |
| `ecommerce` | 쇼핑몰, 예약, 결제 흐름 | 상품/주문/결제 접근성, 명확한 CTA | 전자상거래 약관/환불/개인정보 등 법정 링크 |

## 항상 생성에 반영

- `html lang`, 고유한 `<title>`, viewport
- `<a href="#main" class="skip-to-content">본문 바로가기</a>`
- `header#header`, `main#main`, `footer#footer`
- `main > section > .container`
- 버튼/링크/폼/이미지/테이블의 접근성 기본값
- `var(--color-*)` 색상 토큰, `var(--font-*)` 기본 폰트
- 포커스 표시, 키보드 접근, 터치 영역 44x44px 이상
- 상태 표현은 색상 + 텍스트/아이콘/ARIA를 함께 사용

## 조건부 생성

| 항목 | 생성 조건 | 생성하지 않는 경우 |
|------|-----------|--------------------|
| 공식 배너 | `public-government`이고 정부 상징 사용이 명시됨 | 민간, 사내, 일반 CMS, 기관 정책 미확인 |
| 정부 상징 로고 | 정부 상징 사용 대상 서비스 | 자체 CI/BI가 우선인 민간·사내 프로젝트 |
| 운영기관 식별자 | 상위 운영기관 표시가 과업에 포함됨 | 운영기관 계층이 없거나 브랜드 사이트인 경우 |
| 공공 푸터 필수 링크 | 공공 웹사이트 납품/운영 요구가 있음 | 민간 사이트는 해당 법정/운영 링크로 대체 |
| 검색/로그인/신청/정책 서비스 패턴 | 해당 사용자 여정이 실제 화면에 존재함 | 해당 기능이 없는 화면 |

## 판정 규칙

1. 프로젝트 유형이 명시되어 있으면 그 프로파일을 따른다.
2. 명시가 없으면 `private-corporate` 또는 `admin-cms`로 가정하고 정부 아이덴티티 요소는 생성하지 않는다.
3. 공공기관 프로젝트라도 공식 배너와 정부 상징은 과업지시서, 기관 정책, 운영 주체가 확인된 경우에만 생성한다.
4. 적용 제외 항목은 체크리스트에서 `N/A`로 기록한다.
5. KRDS 패턴을 변형하면 `조건부 PASS`로 사유와 대체 기준을 기록한다.

## LLM 판정 절차

코드 생성 전에 다음 순서로 사이트 유형을 판정한다.

1. 사용자의 명시 표현을 찾는다: 정부, 지자체, 공공기관, 공기업, 산하기관, 민간, 기업, 브랜드, 쇼핑몰, 관리자, CMS, 사내 시스템.
2. 클라이언트/운영 주체를 확인한다: 중앙부처·지자체면 `public-government`, 공기업·출자출연·산하기관이면 `public-affiliated`, 일반 회사면 `private-corporate`, 내부 업무 시스템이면 `admin-cms`.
3. 사이트 목적을 확인한다: 민원/신청/정책/공고/검색 중심이면 공공 서비스 패턴 후보, 상품/예약/결제면 `ecommerce`, 운영/관리/통계면 `admin-cms`.
4. 정부 상징 사용 여부를 별도 판정한다. 공공기관 프로젝트라도 정부 상징 사용이 확인되지 않으면 공식 배너와 정부 상징 로고는 생성하지 않는다.
5. 증거가 부족하면 사용자에게 짧게 확인한다. 단, 구현을 계속해야 하는 상황에서는 정부 아이덴티티 요소를 제외하고 공통 접근성/구조 규칙만 적용한다.

## 판정 예시

| 입력 단서 | 판정 | 생성 방향 |
|-----------|------|-----------|
| "JDC면세점 반응형 홈페이지" | `public-affiliated` 후보 | 공통 품질 + 기관 사이트 헤더/푸터. 공식 배너는 요구 확인 전 생성 금지 |
| "인포마인드 홈페이지" | `private-corporate` | 민간 기업 사이트. 정부 아이덴티티 N/A |
| "CMS inSTACK 관리자" | `admin-cms` | 고밀도 폼/테이블/검색. 정부 아이덴티티 N/A |
| "지자체 민원 신청 서비스" | `public-government` 후보 | 신청 서비스 패턴 적용. 정부 상징/공식 배너는 과업 확인 후 |
| "쇼핑몰 이벤트 페이지" | `ecommerce` 또는 `private-corporate` | 상품/구매 흐름과 법정 링크 중심 |
